1 Introduction  
 
  
Intel’s System Usage Report (SUR) collects performance data on PCs and other 
computation platforms, and then analyzes the results in specific engagements. The SUR 
data is a unique resource, covering millions of real computers worldwide. It reveals 
system configuration and system state information about these computers — and is 
perhaps the best global resource for measuring actual use. The following Data 
Dictionary summarizes the data that SUR collects and uses to perform analysis of 
system telemetries.  
1.1 Summary of Information Analyzed  
The SUR dataset covers the following kinds of “dimensions”, “entities”, and “attributes”:  
System Configuration:   
• 
Model Vendor (OEM)  
• 
Model Name (Market Name)  
• 
CPU  
• 
Memory  
• 
Disk  
• 
Wifi  
• 
Graphics  
• 
Operating System   Geography / Location  
Usage or State:  
• 
Application Usage (Foreground processes)  
• 
Process (Background processes)  
• 
CPU State (C states and P states)  
• 
System power state (S states) 
• 
Battery State  
• 
Thermal state  
• 
Wifi usage  


  
DCA Data Dictionary 
  
10 
• 
Device usage (Monitors, Keyboards, Mice, etc.)  
• 
Disk usage  
There are several levels of metrics in SUR. These levels span from collection of raw 
data to performing analysis on production-level data.  
• 
Collector Level: The raw data collected by the Collector is stored in a SQLite 
database on the host system.  
• 
ATL Level: The Analyzer Task Library (ATL) queries the SQLite database and 
writes the data into flat files. These flat files are then uploaded.  
• 
ETL Level: Using Big Data Technology, the data is uploaded, processed, and 
stored for analysis.

2 Collector Layer Data Dictionary  
 
  
The data that is collected from a system by the Collector is handled by Input Libraries 
(IL). Each IL will store information and/or track system usage based on a specific 
category of input. When a system is used, the ILs will analyze the utilization and then 
store the relevant IL information into a SQLite database. The database is segmented by 
user and system utilization. This section of documentation is divided to list the ILs and 
then examples of inputs that ILs record.  
  
2.1 Input Libraries (IL)  
  
The following is a table of Input Libraries that are included with the SUR Collector:

3 ATL Layer  
 
  
The information collected regarding the system and the user, described in the Collector 
Layer Data Dictionary section, are stored in a SQLite database. The analyzer task library 
(ATL) runs queries on the SQLite database to generate a group of flat files meant for 
remote upload to the cloud. The ATL will group together related metrics into specific 
flat files. Examples of flat file categories are:  
• 
CPU related metrics  
• 
Memory related metrics  
• 
Network usage metrics  
• 
WiFi related metrics  
• 
Mouse cursor state metrics  
• 
Application usage metrics  
• 
OS-Events   
•
